Beacons: Multi-Signal Personalization Engine

üöÄ The Innovation

Beacons was an intelligent alert system that went beyond keyword matching. By integrating proprietary readership data from GovExec's media properties (Defense One, NextGov, Federal Computer Week), we could understand what government buyers and contractors were actually interested in‚Äînot just what they said they wanted.

Signal Capture Architecture

Designed pipeline to ingest readership events from GovExec's analytics platform‚Äîwhich articles each user read, dwell time indicating engagement depth, search queries, and topic following patterns. This created a behavioral signal layer showing implicit interest beyond explicit preferences.

Topic Extraction & Heat Map Generation

Built taxonomy mapping GovExec articles and contract solicitations to common topics (cybersecurity, zero trust, cloud migration, AI/ML). Generated dynamic interest profiles: if a user read 5 articles about zero trust in the past week, signal strength = 0.9 (high). Updated daily as behavior changed.

Multi-Signal Matching Algorithm

Synthesized three context sources: explicit preferences (saved searches, NAICS codes), behavioral signals (readership heat map), and historical patterns (past contract engagement). Composite scoring function weighted: 50% semantic similarity, 30% readership boost, 20% historical pattern match.

Continuous Learning Loop

Captured feedback to improve matching: implicit positive (opened alert, saved contract), implicit negative (dismissed without opening), explicit feedback (thumbs up/down). Iteratively improved signal weighting through A/B testing, achieving 76% relevance rate.

AI Insights: RAG-Powered Intelligence

üîç The Problem

Government contractors were drowning in procurement data‚Äî100,000+ new contracts weekly, each 50+ pages of dense legal and technical language. Finding relevant opportunities required hours of manual research. Users needed natural language query capability: "Who are likely bidders for this DOD cybersecurity RFP?" or "What compliance risks exist in this solicitation?"

üèóÔ∏è RAG Architecture I Built

  • Data Enrichment Pipeline: Processed 100K+ contracts weekly, extracting metadata (NAICS codes, agency classifications, vendor relationships, historical patterns)
  • Embedding Generation: Used OpenAI text-embedding-ada-002 (1536 dimensions) to create semantic representations of contracts
  • Vector Storage: Elasticsearch with cosine similarity indexing for sub-500ms retrieval at scale
  • Context Compression: Reduced 8,000-token solicitations to 1,200 tokens while maintaining >90% accuracy through extractive summarization
  • LLM Integration: OpenAI Chat Completions API with tiered routing (GPT-3.5 for simple queries, GPT-4 for complex analysis)
  • Prompt Engineering: Structured outputs, few-shot examples, chain-of-thought reasoning for premium tier

üìä Evaluation Framework I Established

Built comprehensive measurement system across offline benchmarks, production monitoring, and continuous experimentation:

  • Offline Benchmarks: Created 200+ labeled Q&A pairs, measured retrieval precision/recall, answer relevance (1-5 rubric), hallucination rate
  • Model Selection: Compared GPT-3.5 vs GPT-4 - discovered 40% better accuracy but 10x cost, led to tiered routing strategy
  • Production Monitoring: Power BI dashboards tracking API latency (p50/p95/p99), cost per query, user engagement, hallucination incidents
  • A/B Testing: Semantic chunking (+18% quality), HyDE retrieval (killed due to latency), chain-of-thought (+35% accuracy, premium only)
  • Cost Optimization: Identified inefficient prompts wasting tokens, refactored and saved $8K/month; implemented caching reducing API calls 40%
